Первичные установки
Начинаем новый проект для DLNA Proxy. Для начала сделаем не прокси, а повторитель. То есть задача такая, у меня есть развитая VPN  сеть и в разных сегментах находятся DLNA серверы. Так как протокол работает в том числе используя широковещательные адреса, через VPN мне понадобится делать проксирование из сети в сеть. Однако для первого этапа я хочу создать DLNA повторитель. То есть по сути когда клиент (VLC например) делает запрос на поиск серверов, мой повторитель ждет запрос, и генерирует ответ, как буд то бы присутствует DLNA сервер в сегменте, а данные будет отдавать настоящего DLNA сервера  который находится удаленно. Это должно работать, т.к. мы попросту пока эмулируем работу с широковещательным адресом, а уже потом начинается работа VLC плеера как клиента непосредственно по  TCP - адресно.
Твоя задача не писать самому код в файлы, только по моему запросу ты даешь мне подсказки очень последтвательно. То есть сначала мы расписываем шаги, которые нам нужно прдпринять и каждый шаг (начиная с создания вообще проекта в GO и подключение библиотек, компилирования) все это нужно будет проговорить. Объяснить какие файлы в GO и зачем. Пояснения по синтаксису. То есть по сути это тестовый проект, учебный, благодаря которому я немного начну разбираться в GO и в его синтаксисе, приемах программирования.
Для начала стоит расписать последовательность действий.
Мы исходим из того что у нас появляется широковещательный запрос на поиск DLNA сервера. можно будет его послать непосресдтвенно через консоль на удаленном сервере, посмотреть ответ и потом этот ответ мы тут будем эмулировать.
Начнем с создания проекта в GO. Не беги вперед, не пиши много, максимально коротоко и лаконично. Я тебе буду задавать д ополнительно вопросы и только по ним будем идти. Важно не выдавать много информации сразу, так как я буду сразу ощущать надобность задать вопрос и от ответа будет зависеть многое, в том числе что я попросту не буду читать то что ты написал изначально, т.к. моя мысль ушла уже в другую сторону исходя из данного тобой мне ответа.
Итак, начнем.


1.
Создать Go-модуль Команда будет примерно такая:
go mod init dlna_proxy
Это создаст файл go.mod.
2.
Создать минимальную структуру проекта
На старте достаточно:
dlna_proxy/
├── go.mod
├── .gitignore
└── cmd/
    └── dlna-repeater/
        └── main.go
3.
Понять роль файлов
go.mod
Описание Go-модуля: имя проекта, версия Go, зависимости.

Точка входа в программу. Здесь будет package main и функция main().
.gitignore
Файл для Git, чтобы не сохранять мусор: бинарники, временные файлы IDE и т.п.
4.
Первый рабочий этап
Не эмулируем DLNA сразу.
Сначала программа должна просто слушать SSDP-запросы:
UDP 239.255.255.250:1900
И печатать в консоль то, что пришло.
5.
Второй этап
Отправим реальный запрос поиска DLNA-серверов из консоли и посмотрим, как выглядит ответ настоящего сервера.
6.
Третий этап
Научим нашу программу отвечать на такой запрос вручную сформированным SSDP-ответом.
7.
Четвертый этап
VLC увидит наш повторитель как DLNA-сервер, а в ответе мы укажем адрес настоящего удаленного DLNA-сервера.
